# Git 運用ルール

このファイルは、このリポジトリで作業するときの Git 運用ルールです。

## ブランチ

- デフォルトブランチ: `main`
- 作業ブランチ: `feature/<short-name>`

## コミット

- コミットは小さく、目的を1つに絞る
- コミットメッセージは命令形（例: "Add X", "Fix Y"）
- 可能な限り、リファクタと挙動変更は分ける

## PR 前の確認

- 実行:
  - `go test ./...`
  - `go vet ./...`
  - `goimports -w .`
  - `golangci-lint run`（導入済みの場合）
- `.claude/rules/` 配下の仕様に関わる変更をしたら、仕様ドキュメントも同時に更新する

## 注意

- `.claude/rules/` 配下を仕様の正（SSOT）として扱う
- シークレットやトークンはコミットしない（必要ならプレースホルダ＋説明を書く）
